Project Overview

About the Student Registration System

This project is being built in stages by the Computer Applications team at Innolift Institute of Technology. Day 1 establishes the front-end structure — Flask routing, Jinja templates, and a consistent visual identity across all pages. Database storage, validation, and live record management will be layered on in the days that follow.

D1

Frontend Structure

Flask project setup, page routing with render_template(), and the Home, Registration, Students, and About pages styled with a shared theme.

D2

Database Integration

SQLite is connected so registration data is stored permanently and the Students page reflects real records.

D3

CRUD & Validation

Editing, deleting, and searching student records, plus form validation and user feedback messages.

Features

What this system offers

A

Centralized Registration

One consistent form for capturing every new student's academic and contact details.

B

Departmental View

Records are tagged by department and year, making it easy to scan rosters at a glance.

C

Responsive Layout

A mobile-friendly navigation bar and grid layouts that adapt to any screen size.

D

Built to Extend

Clean Flask routes and templates make it straightforward to plug in SQLite next.
